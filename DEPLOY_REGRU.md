# Инструкция по загрузке MoreDota на хостинг REG.RU

## ШАГ 1: Собрать сайт (на вашем компьютере)

1. Откройте терминал/командную строку в папке проекта
2. Выполните команды:

```bash
pnpm install
pnpm run build
```

После этого появится папка **dist** - это и есть ваш готовый сайт.

---

## ШАГ 2: Подключиться к хостингу REG.RU

### Вариант А: Через FileZilla (рекомендуется)

1. Скачайте FileZilla: https://filezilla.ru/get/
2. Установите и откройте FileZilla
3. Введите данные для подключения (из письма от REG.RU):
   - **Хост:** ftp.вашдомен.ru (или IP-адрес)
   - **Имя пользователя:** ваш логин от REG.RU
   - **Пароль:** ваш пароль от FTP
   - **Порт:** 21
4. Нажмите "Быстрое соединение"

### Вариант Б: Через веб-панель REG.RU

1. Войдите в личный кабинет REG.RU
2. Перейдите в раздел "Хостинг" → "Файловый менеджер"

---

## ШАГ 3: Загрузить файлы на хостинг

1. **Удалите все файлы** из папки `public_html` или `www` на хостинге (если там что-то есть)

2. **Загрузите ВСЕ файлы** из папки **dist** в корневую папку сайта:
   - Если видите папку `public_html` → загружайте в неё
   - Если видите папку `www` → загружайте в неё
   - Если ничего нет → загружайте в корень

3. **Загрузите файл .htaccess** (он лежит в корне проекта)
   - Этот файл может быть скрыт - включите показ скрытых файлов

**ВАЖНО:** Должна получиться такая структура на хостинге:
```
public_html/
  ├── index.html
  ├── .htaccess
  ├── assets/
  │   ├── index-abc123.js
  │   ├── index-xyz789.css
  │   └── ...
  └── vite.svg
```

---

## ШАГ 4: Проверить работу

1. Откройте ваш сайт moredota.com в браузере
2. Если видите сайт - всё работает! ✅
3. Если видите ошибку - проверьте, что:
   - Файлы загружены в правильную папку
   - Файл .htaccess загружен
   - Права доступа к файлам 644, к папкам 755

---

## ШАГ 5: Обновление сайта

Когда нужно что-то изменить:

1. Измените код в проекте
2. Снова выполните: `pnpm run build`
3. Удалите старые файлы на хостинге
4. Загрузите новые файлы из папки dist

---

## ВАЖНО: Админ-панель

Админ-панель (`/admin`) пока работает только для демонстрации.
Чтобы она реально сохраняла данные, нужно подключить базу данных.

Для этого есть 2 варианта:

### Вариант 1: Подключить MySQL от REG.RU
1. В панели REG.RU создайте базу данных MySQL
2. Нужен будет PHP бэкенд для сохранения данных
3. Напишите мне, помогу настроить

### Вариант 2: Подключить Supabase (бесплатно)
1. Зарегистрируйтесь на https://supabase.com
2. Создайте проект
3. Напишите мне, я подключу базу данных к админ-панели

---

## Если что-то не работает

1. Проверьте, что в папке на хостинге есть файл `index.html`
2. Проверьте, что файл `.htaccess` загружен (может быть скрыт)
3. Убедитесь, что загружена папка `assets` со всеми файлами
4. Попробуйте очистить кэш браузера (Ctrl+F5)

Если ничего не помогает - напишите мне, разберёмся!
